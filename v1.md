## PHASE 4: Top Story Hero Card

**Why:** The most visually impactful element. Needs special handling because `top_story` is a string from the backend.

### 4.1 Backend change — Return `top_story` as structured object
Currently: `top_story` stored as `"title: extended_summary"` (string)
**Change `backend/briefing/generator.py`** `store_briefing()`:
- Store `top_story` as JSON object: `{"title": ..., "extended_summary": ..., "url": ..., "source": ..., "why_it_matters": ...}`
- Change `backend/schemas/briefings.py`: `top_story: dict[str, Any] | str = ""`
- Change `backend/api/v1/briefings.py`: when returning detail, parse `top_story` if it's a JSON string

**OR (safer — no migration needed):**
- Keep backend as is (string)
- Parse the string on frontend: split on first `": "` to get title and summary
- This avoids DB migration and is backwards-compatible with existing briefings

**Decision: Parse on frontend** (safer, no backend deploy needed)

### 4.2 Create `TopStoryCard` component — `src/components/briefing/top-story-card.tsx`
```typescript
interface TopStoryCardProps {
  topStory: string;     // "title: extended_summary" from backend
  generatedAt?: string; // briefing timestamp for "X ago" display
}
```

UI (matching mockup):
- Dark gradient background: `bg-gradient-to-br from-slate-800 to-slate-900`
- Rounded 16px: `rounded-2xl`
- Padding 24px: `p-6`
- Decorative blue circle (absolute positioned, top-right, `bg-blue-500/15`)
- Parse top_story string → title (bold, white, `text-lg font-bold`) + summary (slate-400, `text-sm`)
- No impact badge on hero (it's always the top story = highest impact)
- Source info: if parseable, show source dot + source name + time
- **Dark mode:** slightly different gradient (`from-slate-700 to-slate-800`)
- Card is NOT clickable (no URL available from the string format)

### 4.3 Add to briefing page
- Replace the current `top_story` string-in-a-box with `<TopStoryCard>`
- Also replace the `top_story` object rendering path with `<TopStoryCard>` for the object case

### 4.4 Verification
- [ ] Hero card renders with dark gradient background

## Risk Mitigations

| Risk | Mitigation |
|------|------------|
| `top_story` format changes break old briefings | Parse string format on frontend, handle both string and object |
| Source key not in map | Fallback: capitalize key, grey dot |
| Dark mode contrast issues | Audit every component in dark mode (Phase 10) |
| PWA caching stale CSS | Static sw.js clears caches; Vercel deploys with new hashes |
| Section pills overflow on small screens | Horizontal scroll with hidden scrollbar |
| Touch targets too small | Enforce min 44px on all interactive elements |
| Build fails after changes | Run `npm run build` before committing (Phase 14) |
| Tests break | Run tests after each phase, fix immediately |
